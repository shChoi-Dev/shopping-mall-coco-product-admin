# 🛍️ Coco Shopping Mall (포트폴리오)
**"사용자 후기 중심 뷰티 커머스 플랫폼"**

> **프로젝트 개요**
> - **수행 기간**: 2025.10 ~ 2025.12 (2개월)
> - **참여 인원**: 풀스택 6명 
> - **담당 역할**: 상품 및 관리자 도메인 설계, 전역 에러 핸들링, 서비스 성능 및 UX 고도화
> - **기술 스택**: Java 17, Spring Boot 3.5, Spring Security, Oracle, React, AWS EC2, Nginx

## 💡 나의 핵심 기여 및 성과 (Key Contributions)

### 1. 🚀 성능 최적화: 메인 페이지 로딩 속도 90% 개선
서비스 특성상 고용량 상품 이미지가 다수 노출되어, AWS EC2 배포 후 메인 페이지 로딩에 **7초 이상 소요되는 병목 현상**이 발생했습니다.
- **해결 과정**: 
  - 압축률이 높은 WebP 도입을 검토했으나 서버 환경 제약 및 라이브러리 호환성 문제로 난항을 겪었습니다. 개발 지연을 막고 사용자 불편을 즉각 해소하기 위해 **Thumbnailator 라이브러리를 활용한 우회 최적화**를 적용했습니다.
  - 테스트를 거쳐 시각적 품질 저하가 없는 최적의 화질(0.8) 및 해상도 제한을 설정하여, 원본 6MB 이상의 이미지를 **1MB 이하(약 85% 감소)** 로 압축했습니다.
- **결과**: 메인 페이지 로딩 속도를 **1초 이내로 단축(기본 대비 90% 향상)** 하여 쾌적한 쇼핑 환경을 구축했습니다. 제약 속에서도 유연하게 대안을 찾아 성능을 끌어올린 실무적 문제 해결 경험입니다.

### 2. 🛡️ 에러 핸들링 및 프론트엔드 UX 고도화
사용자 관점에서 불편함이 없는, 안전하고 효율적인 프로그램을 제공하기 위해 UI/UX 마감 작업과 에러 핸들링에 집중했습니다.
- **전역 인증/에러 핸들링**: 토큰 만료(401) 시 API 단에서 요청을 가로채어 안전하게 로컬 데이터를 삭제하고 로그인 페이지로 유도하는 로직을 구현했습니다.
- **사용자 친화적 예외 페이지**: 404(Not Found), 403(Forbidden), 상품 판매 중지(ProductStopped) 등 예외 상황 발생 시 빈 화면이 아닌 **직관적인 에러 UI**로 라우팅하여 사용자 이탈을 방지했습니다.
- **프론트엔드 완성도 제고**: 미구현되었거나 비어있는 컴포넌트를 보완하여 전체적인 서비스의 완성도를 실무 수준으로 끌어올렸습니다.

### 3. ⚙️ 상품 및 관리자(Admin) 도메인 구축
- **상품 관리 API 설계**: 다중 이미지 업로드 및 카테고리별 상품 조회, 검색, 필터링 기능 구현
- **관리자 전용 대시보드**: 상품 등록/수정/삭제(CRUD), 카테고리 계층 관리, 전체 회원 및 주문 현황 통계 조회 API 구현

### 4. 🤝 팀 프로세스 개선: DTO 네이밍 컨벤션 정립
단순한 코드 작성을 넘어, 팀의 협업 효율을 저해하는 근본적인 원인을 파악하고 프로세스를 개선했습니다.
- **문제 인식**: 프로젝트 후반, 주문 도메인 담당 팀원이 클라이언트-서버 간 데이터 바인딩 오류로 개발에 난항을 겪음.
- **원인 분석 및 해결**: 함께 디버깅을 진행하여 프론트엔드의 `CartQty`와 백엔드의 `OrderQty` 등 **DTO 변수명 불일치**가 원인임을 규명하고 즉시 오류를 수정했습니다.
- **팀 룰 제안**: 재발 방지를 위해 긴급 회의를 소집하여, **API 명세서 최신화 및 프론트-백엔드 간 네이밍 컨벤션 통일**을 제안했습니다. 이를 통해 후반부 연동 에러 발생률을 크게 낮출 수 있었습니다.

## 🛠️ 내가 작성한 주요 코드 (Tech Details)

### [1] 이미지 리사이징 및 압축 최적화 (Backend)
> **`AdminProductService.java`**
> 고용량 이미지로 인한 로딩 지연을 해결하기 위해 업로드 시점에 이미지를 최적화합니다. 환경적 제약으로 WebP 적용이 불가했던 상황에서, Thumbnailator를 활용해 해상도와 품질(Quality 0.8)을 조정하는 차선책을 적용하여 용량을 대폭 줄였습니다.

```java
// 💡 고용량 원본 이미지를 그대로 저장하지 않고, Thumbnailator를 통해 리사이징 및 압축 수행
// WebP 전환의 대안으로, 시각적 손실이 거의 없는 0.8 품질의 JPG 압축을 선택하여 로딩 속도 개선
String baseName = originalFilename.substring(0, originalFilename.lastIndexOf("."));
String savedFileName = UUID.randomUUID().toString() + "_" + baseName + ".jpg"; 
File dest = new File(getProductUploadPath() + savedFileName);

try {
    Thumbnails.of(file.getInputStream())
        .size(1000, 1000)      // 최대 너비/높이를 1000px로 제한하여 해상도 최적화
        .outputFormat("jpg")   // 서버 호환성이 높은 JPG 포맷으로 통일
        .outputQuality(0.8)    // 이미지 품질 80% 설정 (용량 1MB 이하로 약 85% 감소 달성)
        .toFile(dest);
} catch (IOException e) {
    throw new RuntimeException("이미지 압축 및 저장 중 오류 발생", e);
}
```

### [2] 전역 인증 예외 처리 및 직관적 에러 라우팅 (Frontend)
> **`api.js` & `App.js`**
> API 통신 중 발생하는 인증 만료 및 권한 오류를 공통 함수에서 처리하여 프론트엔드 코드의 중복을 제거하고, 예외 발생 시 사용자를 튕기지 않고 알맞은 안내 페이지로 부드럽게 유도(UX 개선)했습니다.

```javascript
// api.js : JWT 토큰 기반 API 통신 및 401(Unauthorized) 전역 핸들링
export const fetchWithAuth = async (url, options = {}) => {
  const headers = getAuthHeaders();
  const response = await fetch(`${API_BASE_URL}${url}`, { ...options, headers });

  // 토큰 만료 또는 인증 실패 시 사용자 로컬 데이터 정리 후 로그인 페이지로 리다이렉트
  if (response.status === 401) {
    logout(); 
    window.location.href = '/login';
    throw new Error('인증이 만료되었습니다.');
  }
  return response;
};
```

```javascript
// App.js : 상황별 커스텀 에러 페이지 라우팅 적용
<Routes>
  {/* 정상적인 비즈니스 로직 라우트들... */}
  
  {/* 예외 상황 발생 시 빈 화면 대신 사용자 친화적인 에러 페이지 제공 */}
  <Route path="/error/403" element={<Forbidden />} />           {/* 접근 권한 없음 */}
  <Route path="/product-stopped" element={<ProductStopped />} /> {/* 판매 중지된 상품 */}
  <Route path="*" element={<NotFound />} />                      {/* 잘못된 URL 접근 */}
</Routes>
```
